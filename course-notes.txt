8. Big O: Intro  +  9. Big O: Worst Case

Big O Notation
    Describes how the runtime or memory usage of an algorithm grows as the input size grows.
    Used to compare algorithm efficiency, especially when working with large datasets.

Time Complexity - A measurement based on 'number of operations' that it takes to 
                  complete something
                  *Number of operations/steps
                        Lower is faster
                  How fast the program runs
                  Does not measure time in seconds/milliseconds because this unit can be
                  dependent on system speed
                  Counting loops, recursive calls, and basic operations

Space Complexity - The total amount of memory or storage space required by an algorithm to run until completion
                   *Memory units (RAM/Storage)
                        Lower is better
                        Lower is efficient
                        Less memory usage
                   How much memory the program uses
                        input variables
                        auxiliary space / extra memory from temporary data or recursion stacks
                   Counting variables, data structures, and the recursion stack

*Optimizing for one requires sacrificing the other

Big O,  Greek Letters
    - Omega    -->   Best Case (least operations)
                        Big O(Ω) 
                        * Alt + 234 
    - Theta    -->   Average Case
                        Big O(Θ)    
                        * Alt + 233 
    - Omicron  -->   Worst Case
                        Big O(Ο) 
                        * Capital O

* "Technically, there is no 'average case' or 'best case' for 'Big O' but some people 
    will still use these greek letters
* 'Big O' is always worst case.

10. Big O: O(n)

O(n)
    Reads as:
        'O' of 'n'
    O(n) means linear time complexity
    It is always 'Proportional'
    This can be seen in loops
    looping code,
        n = 5,  about 5 operations
        n = 1000,  about 1000 operations

11. Big O: Drop Constants

Rules of Simplification
    OR
Simplification Rules:
    1) Drop Constants
        Ignore fixed numbers when describing algorithm growth

Don't write this,   O(2n)
Keep it simple,     O(n)
                    *Drop the Constant

This keeps O(n) distinct from other Big O notation that involves exponentially larger Time Complexity
    such as O(n²).

12. Big O: O(n²)

O(n²)
    'O of n squared'
    Also called, 'quadratic time complexity' 
    It means that an algorithm’s running time grows proportionally to the square of 'n'
    It usually happens in nested loops
        The outer loop runs 'n' times
            the inner loop runs 'n' times
        n * n = n²
    Time Complexity increases VERY fast
    O(n²) algorithms become slow with large datasets
    If you have 1,000 items:
        O(n) ≈ 1,000 operations
        O(n²) ≈ 1,000,000 operations

14. Big O: O(1)

O(1)
    'O of one'
    Called, 'constant time complexity'
    It means the algorithm takes the same amount of time no matter how large the input becomes
    The most efficient Big O

    n + n =       O(2)
    n + n + n =   O(3)

15. Big O: O(log n)

O(log n)
    'O log of n'
    Called 'Logarithmic time complexity'
    Often refer as, 'Divide and Conquer'
    It means the algorithm gets rid of a large portion of the data each step instead of 
        checking every item one by one.
    For 1 million items:
        O(n) may need 1,000,000 checks
        O(log n) needs about 20 checks

Logarithmic algorithms are considered very fast for searching large datasets

----

O(n log n)
    'O of n log n'
    This algorithm performs:
        'n' work
        combined with logarithmic splitting or processing
    This is the most efficient sorting algorithms use it
    It is one of the most important time complexities

17. Big O: Vectors

Talking points below can apply to arrays

Adding or removing an element from the 'End' of a dataset is O(1) (constant time complexity)
    this is very simple and requires the least operations

Adding or removing an element from the 'Front' of a dataset is O(n) (linear time complexity)
    because the 'index' of each element after it has to be shifted back by 1
    *It will remain as O(n) if we add/remove an element at a specific index

Finding an element value is O(n) because we have to look at the value at each index

Finding an element by index O(1) because we know the memory address/location

19. Pointers

Notable Behavior between Pointers and Normal Variables
    - Pointers point to a memory addresss
        - * (de-reference) is used to access to the stored value at the memory address
        - pointers can also point to another memory address (pointer)
    - Normal Variables only know about a stored value

    Normal:
        int num1 = new 11;
        int num2 = num1;
        num1 = 22;
        // psuedo code: print num1 & num2

        output:
            num1 prints 22
            num2 prints 11

    Pointers:
        int* num1 = new int(11);
        int* num2 = num1;
        *num1 = new int(22);
        // psuedo code: print num1 & num2

        output:
            num1 prints 22
            num2 prints 22

    *num2 points to the memory address of num1 and de-referencing num2 will return the stored value 
         at the memory address of num1

21. Linked List: Intro

Talking Points
    - Linked List properties
    - Vectors allows direct and quick access to an 'index' in its data set
    - 'Green Squares' represent Vectors/arrays
    - 'Purple Circles' represent Nodes such as Linked Lists
    - Contiguous and Non-Contiguous
    - Each node should have a 'next' property pointing to the next node in the linked list

Linked Lists & Terminology
    Head - Stores memory address of the 'First' node in a list.
            "Head points to the first node"
    Tail - Stores memory address of the 'Last' node in a list.
            Important: the 'Tail' node points to 'nullptr' and this is how we know 
                        that a node is the 'Tail'
    Non-Contiguous - Each node is not spatially organized next to each other in memory.
                        This is different from vectors/arrays where the entire data set is contiguous.

22. LL: Big O 

Linked List Talking Points 
    - Adding node to End 
        - O(1)  Constant Time 
        - this context: 'n' = last node 
        - Process
            1) Set the original node(n)->'next' to point to new 'node(n)'
            2) Set 'Tail' to point to the new node(n) in the linked list 
    - Removing nodes from End 
        - O(n)  Linear Time 
        - Process 
            1) Remove last node by setting node(n-1)->'next' to 'nullptr'
                - Now this is the last node(n)
            2) Find the node(n-1) that points to the last node(n) in the linked list 
                - Iterate through entire Linked List in order to find the node(n-1) that points to 
                    the last node(n) in the list 
            3) Reassign 'Tail'->'next' to 'node(n-1)'->'next'
                - Now 'node(n-1)' and 'Tail' points to the very last node(n) in the linked list 

    - Adding node to Front
        - O(1)  Constant Time
        - this context: 'i' = first node
        - Process
            1) Assign new first node(i)->'next' to 'Head' which points to original first node(i+1)
    - Removing node from Front
        - O(1)  Constant Time
        - Process
            1) Set 'Head->next' equal to 'node(i+1)'
                - original node(i) is removed from the linked list
                - now 'node(i+1)' becomes 'node(i)' of the linked list
            
    - Adding node to Middle
        - O(n)  Linear Time
        - clarity: add after node
        - this context: 
            - 'x' = target node to search (in the middle)
            - 'y' = new node (node to be added)
        - Process
            1) Start at 'Head' and Iterate through the linked list to find node(x)
            2) Set 'node(y)->next' point to 'node(x)->next'
                - both 'node(y)' & 'node(x)' point to the same node
            3) Set 'node(x)->next' point to 'node(y)->next'
                - node(x) points to node(y)

    - Searching a node by index
        - O(n)
        - Process 
            1) Start at 'Head' node and Iterate through linked list while accumulating an index

    - Searching a node by value
        - O(n)
        - Process
            1) Iterate through each node and compare the stored value

    - Vector/Array vs Linked List
        - Vector is faster at
            - Removing Last node  O(n)
            - Searching by index  O(n)
        - Linked List is faster at
            - Prepending          O(n)
            - Removing First node O(n)

27. LL: Destructor

LinkedList class has a written Destructor
    - LinkedList default Destructor will delete 'head' and 'tail' (stack objects)
    - LinkedList class does not know how to delete all stored 'Nodes'
        - Memory Leak if 'Nodes' are not deleted properly
    - Process deletion of all 'Node' objects in ~LinkedList() Destructor
        1) Create a temporary Node pointer and set to 'head'
                Node* temp = head;
        2) Iterate each node in the LinkedList while 'head' != nullptr
        3) Set 'head' to 'head->next'
            - Move the 'head' forward
        4) delete 'temp'
            - This deletes the 'Node' in the Heap
        5) Reassign 'temp' = head;
        6) Repeat from step 2
            - Eventually, 'head' will be set to 'nullptr' and race us
                out of the while loop
             
Node class has no written Destructor
    - Default destructor will be called, when 'delete node' executes

~LinkedList() {
    Node* temp = head;
    while(head){
        head = head->next;
        delete temp;
        temp = head;
    }
}

bool insert(int index, int value) {
            
    if (index < 0 || index > length) return false;
    
    if (index == 0) {       // Edge Case
        prepend(value);
        return true;
    }
    if (index == length) {  // Edge Case
        append(value);
        return true;

    temp->next = newNode;
    length++;
    
    return true;
}

 Use 'get()' to return a 'Node*'

Process: Delete a node in the middle
    1) 'temp' Node to store node that will be deleted
    2) 'prevNode' to store node behind 'temp' Node
    3) Set 'prevNode->next' = 'temp->next'
    4) delete 'temp'
    5) decrement length

Edge Case: Index cannot be out of bounds
    1) index < 0
    2) index >= length

Edge Case: Index == 0
    1) return deleteFirst()

Edge Case: Index == length - 1
    1) return deleteLast()

void deleteNode(int index) {
            
    if (index < 0 || index >= length) return;
    if (index == 0) {
        deleteFirst();
        return;
    }
    if (index == length) {
        deleteLast();
        return;
    }
    
    Node* prevNode = get(index - 1);        
    Node* temp = prevNode->next;    // Don't call get() twice | expensive
    
    prevNode->next = temp->next;
    delete temp;                    // Delete the node at 'index'
    length--;
}

void reverse() {
            
    if (length == 1) return;    // Handle Edge Case
    
    Node* temp = head;
    head = tail;
    tail = temp;
    
    Node* after = nullptr;
    Node* before = nullptr;
    
    // Iterate through the list using 'temp', while 'before' trails behind and becomes the reversed 
    //  portion, and 'after keeps track of the next node.
    for (int i = 0; i < length; i++) {
        after = temp->next;     // Cache: Store next node for next iteration
        temp->next = before;    // Reverse: Point the 'next' node to the 'before' node
        before = temp;          // Advance: Move 'before' forward | it now represents the head of the reversed portion.
        temp = after;           // Advance: Move 'temp' to the next node we cached earlier.        
    }
}

Floyd's cycle-finding algorithm (also known as the "tortoise and the hare" algorithm) 
    This algorithm uses two pointers: a slow pointer and a fast pointer. The slow pointer 
    moves one step at a time, while the fast pointer moves two steps at a time. If there 
    is a loop in the linked list, the two pointers will eventually meet at some point. 
    If there is no loop, the fast pointer will reach the end of the list.

bool hasLoop() {
            
    Node* slow = head;
    Node* fast = head;
    
    while (fast && fast->next) {
        slow = slow->next;
        fast = fast->next->next;
        
        if (slow == fast) {
            return true;
        }
    }
    
    return false;
}

        Node* current = head; // Starts at head of list

                values.insert(current->value);
                
                // Update previous to current node
                previous = current;
                
                // Move to next node
                current = current->next;
            }
        }

        dummy = nullptr;
    }

    Node* get(int index) { 
    
        if (length == 0) return nullptr; 
        if (index < 0 || index >= length) return nullptr; 

        bool forward = index < length / 2;  // traversal direction

56. DLL: Palindrome Checker

Palindrome:
    The values in the first-half are the same as the values in the second-half of the list.
    If there are an odd number of elements in the list then the middle value is ignored.

A single number is considered a Palindrome:
    0 is Palindrome
    1 is Palindrome
    9 is palindrome

EMPTY List
    An empty list is considered a palindrome.

Palidrome example: 1 2 4 2 1

Consider the length of the list

Rough thought process:
    - return bool
    - check edge cases
        - length == 0
        - length == 1
        - if any of the above are true, then return true
    - Compute number of iterations required
        - example, length = 5
                   it = 5/2 = 2
                   * 5/2 == 2.5, but the .5 is truncated
    - create 2 temporary nodes
        1) 'left' = head,  traverse forwards towards middle
        2) 'right' = tail, traverse backwards towards middle

    bool isPalindrome() {

        if (length <= 1) return true;

        Node* left = head;
        Node* right = tail;

        int it = length/2;
        while (it > 0) {
            if (left->value != right->value) return false;
            left = left->next;
            right =right->prev;
            it--;
        }

        return true;
    }

    void partitionList(int x) {
        if (head == nullptr) return;

        Node dummy1(0);
        Node dummy2(0);
        Node* prev1 = &dummy1;
        Node* prev2 = &dummy2;
        Node* current = head;

        while (current != nullptr) {
            if (current->value < x) {
                prev1->next = current;
                current->prev = prev1;
                prev1 = current;
            } else {
                prev2->next = current;
                current->prev = prev2;
                prev2 = current;
            }
            current = current->next;
        }

        prev2->next = nullptr;
        prev1->next = dummy2.next;
        if (dummy2.next) {
            dummy2.next->prev = prev1;
        }
        head = dummy1.next;
    }

Instructor algorithm:
    void partitionList(int x) {
        // If the list is empty, do nothing
        if (!head) return;
    
        // Create two dummy nodes to start the two partitions
        Node* dummy1 = new Node(0);  // For values < x
        Node* dummy2 = new Node(0);  // For values >= x
    
        // prev1 and prev2 will build the two lists
        Node* prev1 = dummy1;
        Node* prev2 = dummy2;
    
        Node* current = head;
    
        while (current != nullptr) {
            Node* nextNode = current->next;  // Save the next node
    
            // Disconnect the node from the list
            current->next = nullptr;
            current->prev = nullptr;
    
            if (current->value < x) {
                // Append to the 'less than x' list
                prev1->next = current;
                current->prev = prev1;
                prev1 = current;
            } else {
                // Append to the 'greater or equal to x' list
                prev2->next = current;
                current->prev = prev2;
                prev2 = current;
            }
    
            current = nextNode;
        }
    
        // Terminate the greater-or-equal list
        prev2->next = nullptr;
    
        // Connect the two lists
        prev1->next = dummy2->next;
    
        if (dummy2->next != nullptr) {
            dummy2->next->prev = prev1;
        }
    
        // Update the head of the full list
        head = dummy1->next;
        if (head) head->prev = nullptr;
    
        // Clean up
        delete dummy1;
        delete dummy2;
    }

My algorithm:
    void reverseBetween(int n, int m) {
        if (!head) return;

        Node* dummy = new Node(0);
        dummy->next = head;
        Node* prev = dummy;

        for (int i=0; i < n; i++){
            prev = prev->next;
        }

        Node* current = prev->next;
        
        for (int i=0; i < m - n; i++) {
            dummy->prev = prev->next;   // use 'dummy->prev' as a temporary cache

            // Swap 'next' properties
            prev->next = current->next; 
            current->next = current->next->next;
            prev->next->next = dummy->prev;  //*Set to old prev->next

            // Swap 'prev' properties
            if (current->next) {
                current->next->prev =  current;
            } else {
                current->next = nullptr; // tail = current; // no 'tail' property for this class 
            }
            dummy->prev->prev = prev->next;
            prev->next->prev = prev;
        }

        head = dummy->next;
        head->prev = nullptr;
        delete dummy;
    }

Instructor's algorithm:
    void reverseBetween(int startIndex, int endIndex) {
        // Return early if list is empty or no range to reverse
        if (head == nullptr || startIndex == endIndex) return;
    
        // Dummy node simplifies edge case handling
        Node* dummy = new Node(0);
        dummy->next = head;
        head->prev = dummy;
    
        // Traverse to node BEFORE start index
        Node* prev = dummy;
        for (int i = 0; i < startIndex; i++) {
            prev = prev->next;
        }
    
        // 'current' is the start of the sublist to reverse
        Node* current = prev->next;
    
        // Reverse using node splicing
        for (int i = 0; i < endIndex - startIndex; i++) {
            Node* nodeToMove = current->next;
    
            // Detach nodeToMove
            current->next = nodeToMove->next;
            if (nodeToMove->next) {
                nodeToMove->next->prev = current;
            }
    
            // Insert nodeToMove after prev
            nodeToMove->next = prev->next;
            prev->next->prev = nodeToMove;
            prev->next = nodeToMove;
            nodeToMove->prev = prev;
        }
    
        // Update head in case it changed
        head = dummy->next;
        head->prev = nullptr;
    
        delete dummy;
    }

Notable algorithm difference:
    Mine:

        - Setting all the 'next' properties first forces the following code
            to be rewired in a less safe way
        - Best to detach the right-side(current->next) Node in front of 'prev' Node 
            and then perform rewiring of 'next/prev' pointers

    Implement a member function called swapPairs() that swaps the values of every two 
    adjacent nodes in the list.

    Note: This DoublyLinkedList does not have a tail pointer which will make the implementation 
    easier.

    Input:
        The function is a member of the DoublyLinkedList class, which has a head and a tail pointer, 
        as well as a length attribute.

    Output:
        No explicit output is returned. However, the function should modify the doubly linked list 
        such that the values of every two adjacent nodes are swapped.

    Constraints:
        "The doubly linked list may be empty, have only one node, or have two or more nodes."

    Example:
    Consider the following doubly linked list:

        Head: 1
        Tail: 6
        Length: 6
        
        Doubly Linked List:
        1 <-> 2 <-> 3 <-> 4 <-> 5 <-> 6

        After calling swapPairs(), the list should be:

        Head: 2
        Tail: 5
        Length: 6
        
        Doubly Linked List:
        2 <-> 1 <-> 4 <-> 3 <-> 6 <-> 5

    Consider the following doubly linked list:

        Head: 1
        Tail: 5
        Length: 5
        
        Doubly Linked List:
        1 <-> 2 <-> 3 <-> 4 <-> 5

        After calling swapPairs(), the list should be:

        Head: 2
        Tail: 4
        Length: 5
        
        Doubly Linked List:
        2 <-> 1 <-> 4 <-> 3 <-> 5

    Note that the pointers to the nodes themselves are updated to achieve 
    the swapping of adjacent nodes.  Do not update the values of the nodes.
***

My algorithm:
    void swapPairs() {
        Node* dummy = new Node(0);
        dummy->next = head;

        Node* left = dummy;
        Node* middle = head;

        while (middle && middle->next) {
            Node* right = middle->next;

            left->next = right; // this also changes 'dummy->next' = 'right' on first iteration
            middle->next = right->next;
            right->next = middle;
            if (middle->next) middle->next->prev = middle;
            middle->prev = right;
            right->prev = left;
            
            left = middle;
            middle = middle->next;
        }

        head = dummy->next;
        head->prev = nullptr;
        delete dummy;
        dummy = nullptr;
    }

Instructor's algorithm:
    void swapPairs() {
        Node dummyNode(0);
        dummyNode.next = head;
        Node* previousNode = &dummyNode;
    
        while (head != nullptr && head->next != nullptr) {
            Node* firstNode = head;
            Node* secondNode = head->next;
    
            previousNode->next = secondNode;
            firstNode->next = secondNode->next;
            secondNode->next = firstNode;
    
            secondNode->prev = previousNode;
            firstNode->prev = secondNode;
            if (firstNode->next != nullptr) {
                firstNode->next->prev = firstNode;
            }
    
            head = firstNode->next;
            previousNode = firstNode;
        }
    
        head = dummyNode.next;
        if (head) head->prev = nullptr;
    }

Notable Comparisons:
    Both:
        - are O(n) time and O(1) space
        - both are readable
    mine:

                    Node dummy(0);
                    dummy->next = head;
                    *This is safest

        - No heap allocation

        - Updates 'head' at the end of every iteration

59. Stack: Intro

Stack Talking Points 
    - properties
        - 'next' points to the element below itself
        - the element at the very bottom points to nullptr
        - 'top' points to the item at the very top of list
            - equivalent to 'head' for linked lists
            - there is no 'tail' or 'bottom' property
        - 'height' property stores the number of elements in the stack
            - equivalent to 'length' for linked lists
    - Analogy
        - Stack of plates
        - First plate at the very top is the first plate that can be accessed
        - Cannot get any plates below until the very top plate is removed
    - FIFO 
        - Last-In  First-Outer
        - Last item placed into a list is the First item to be removed
    - pop - key terminology
        - 'pop' - the item at the top of the stack is removed
        - 'pop' off
    - vector - data type
        - std::vector
        - Not fundamentally a stack structure
        - A dynamic array
        - Provides all necessary functionality to behave exactly like a 'stack'
        - common methods
            - push_back(), adds an item to the top of stack
            - pop(), removes item from from the top of stack
    - Adding/Removing items is O(1)
    - Stacks add/remove items from the same end
    - The element/item at the very bottom
